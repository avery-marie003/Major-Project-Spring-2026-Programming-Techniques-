Purpose of the program:

This program is a Point of Sale (POS) System created for the Best Buy Retail Store. The purpose of the program is to help a cashier process customer purchases more easily and accurately.

The system allows the cashier to:
View products with prices and stock levels
Add items to a shopping cart
Remove items from the cart
Calculate subtotal, discount, tax, and total cost
Accept customer payment and calculate change
Print a receipt after checkout
Handle multiple customers in one session

How to run the program:

Make sure Python 3 is installed on your computer
   If not installed, download it from python.org and install it.

2. Save the program file as: pos_system.py

3. Open the folder where pos_system.py is saved.

4. Run the program using Command Prompt/Terminal
Open Command Prompt (Windows) or Terminal (Mac/Linux)
Navigate to the folder where the file is saved
Type: python pos_system.py
Press Enter to run the program
Follow the on‑screen menu instructions to use the POS system.
To stop the program, select the Exit option from the menu.
The program will continue running until the user selects the Exit option.

Assumptions

All prices are in Jamaican Dollars (JMD) 
It is assumed that all product prices, totals, discounts, and taxes shown in the system are calculated and displayed using Jamaican Dollars.

The user enters valid numeric values when required
The system assumes that the cashier enters numbers when prompted for quantities, prices, and payment amounts. For example, it assumes the user will not enter letters where numbers are expected.

Product names are entered correctly (case‑insensitive)
It is assumed that the user enters valid product names that exist in the system. The program converts input to lowercase, so capitalization does not affect product matching.

Discounts are applied before tax
The system assumes that any applicable discount is calculated first and then sales tax is applied to the discounted subtotal, following standard retail practice.

Limitations

The system does not save data to a file or database
All product and transaction data exist only while the program is running. Once the program is closed, all information is lost.

Stock resets when the program is restarted
Because the system does not store data permanently, product stock levels return to their original values each time the program is restarted.

There is no user login or security system
The system does not include cashier login, passwords, or user roles. Any user who runs the program can access all features.

The system does not handle input errors like letters being entered instead of numbers 
If the user enters invalid input (for example, typing letters instead of numbers), the program may display an error or stop running.
